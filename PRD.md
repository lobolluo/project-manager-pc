## 1. 产品概述

### 1.1 产品愿景

ProjectFlow 是一款面向 5-20 人小团队的轻量级项目管理工具，提供看板、任务追踪、团队协作等核心功能，帮助团队高效管理项目进度。

### 1.2 目标用户

| 用户类型 | 特征描述 | 核心需求 |
|----------|----------|----------|
| 项目经理 | 管理多个项目，关注整体进度 | 项目概览、进度追踪、成员管理 |
| 开发工程师 | 执行具体任务，关注待办和优先级 | 看板视图、任务管理、状态流转 |
| 设计师 | 参与多个项目，关注协作 | 任务分配、截止日期提醒 |
| 团队负责人 | 关注团队效率和资源分配 | 统计报表、成员工作量 |

### 1.3 竞品分析

| 产品 | 定位 | 优势 | 不足 |
|------|------|------|------|
| Trello | 看板工具 | 简单易用 | 功能单一，无报表 |
| Notion | All-in-one | 灵活强大 | 学习成本高 |
| Jira | 企业级项目管理 | 功能全面 | 过于复杂，价格高 |
| **ProjectFlow** | 轻量级项目管理 | 简洁 + 看板 + 报表 | MVP 阶段功能精简 |

### 1.4 成功指标

| 指标类别 | 具体指标 | 目标值 |
|----------|----------|--------|
| 功能 | 核心页面可交互 | 4 个页面全部可操作 |
| 性能 | 页面加载时间 | < 2s |
| 兼容性 | 浏览器支持 | Chrome / Firefox / Edge |
| 数据 | 本地持久化 | localStorage 保存 |

## 6. 用户场景

### 场景 1：查看项目概览

- **角色**：项目经理
- **前置条件**：已登录，有项目数据
- **操作流程**：打开应用 → 首页仪表盘 → 查看统计卡片 → 查看周进度图 → 查看最近活动
- **预期结果**：一目了然掌握所有项目状态

### 场景 2：管理任务状态

- **角色**：开发工程师
- **前置条件**：有待办任务
- **操作流程**：进入看板页 → 拖拽任务卡片从"待办"到"进行中" → 状态自动保存
- **预期结果**：任务状态更新，侧边栏/面包屑正常

### 场景 3：创建新任务

- **角色**：团队成员
- **前置条件**：在看板页或项目详情页
- **操作流程**：点击"新建任务" → 填写标题/优先级/截止日期 → 保存
- **预期结果**：任务出现在看板"待办"列

## 7. 权限与角色

| 角色 | 权限范围 | 说明 |
|------|----------|------|
| manager | 全部操作 | 项目经理，可管理项目和成员 |
| developer | 任务 CRUD | 开发工程师，管理自己的任务 |
| designer | 任务查看 + 编辑 | 设计师，查看和编辑分配的任务 |
| tester | 任务查看 | 测试工程师，查看任务状态 |
