### P-1. 安装 AWS CLI

**在哪**：终端
**操作**：复制粘贴运行

```bash
brew install awscli
```

验证安装成功：
```bash
aws --version
# 应输出类似：aws-cli/2.x.x
```

- [ ] 已完成

---

### P-2. 注册 AWS 账户

**在哪**：浏览器
**操作**：

1. 打开 https://aws.amazon.com/free/
2. 点「创建 AWS 账户」
3. 填写邮箱 → 验证码 → 设置密码
4. 选择「个人」账户类型
5. 填写地址信息（可用 GPT 生成美国地址）
6. 绑定信用卡（推荐 Bybit Card / Roogoo）
7. 完成手机验证
8. 选择「基本支持 - 免费」计划

注册完成后，账户有 **$100** 信用额度 + Free Tier 权益。

- [ ] 已完成

---

### P-3. 创建 AWS Access Key

**在哪**：AWS 控制台
**操作**：

1. 登录 https://console.aws.amazon.com/
2. 右上角用户名 → 「安全凭证」
3. 左侧选「访问密钥」
4. 点「创建新的访问密钥」
5. 选择「命令行界面 (CLI)」→ 勾选确认 → 下一步
6. **立即下载 CSV 或复制保存**（Secret Key 只显示一次！）

记录下来：
```
Access Key ID:     AKIAxxxxxxxxxxxxxxxx
Secret Access Key:  xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

- [ ] 已完成

---

### P-4. 配置 AWS CLI

**在哪**：终端
**操作**：复制粘贴运行，按提示填入上一步的 Key

```bash
aws configure
```

依次输入：
```
Access Key ID:      <粘贴 P-3 获取的 ID>
Secret Access Key:   <粘贴 P-3 获取的 Secret>
Default region name: ap-southeast-1
Default output format: json
```

验证：
```bash
aws sts get-caller-identity
# 应输出你的 AWS 账户信息，不报错即成功
```

- [ ] 已完成

---

### P-5. 登录 Stitch 获取 API Token

**在哪**：浏览器
**操作**：

1. 打开 https://stitch.withgoogle.com/
2. 用 Google 账号登录
3. 进入 Settings 页面
4. 找到 API Token / Developer 设置
5. 复制 Token 保存

```
Stitch API Token:  <粘贴你的 token>
```

- [ ] 已完成

---

### P-6. 配置 Stitch MCP

**在哪**：终端（Claude Code 内执行）
**操作**：复制粘贴以下命令

```bash
claude mcp add stitch --transport http https://stitch.googleapis.com/mcp --header "X-Goog-Api-Key: <粘贴 P-5 获取的 API Token>" -s user
```

> ⚠️ Stitch 是 Google 的远程 HTTP MCP 服务，不是本地 stdio。必须用 `--transport http` 方式。

**验证**：重启 Claude Code 后运行 `/mcp`，确认 `stitch` 显示 ✓ Connected。

- [x] Stitch MCP 已配置（HTTP 远程）

## Step 3：AWS 部署（AWS CLI）

### 3-1 让 Claude Code 用 AWS CLI 部署

**在哪**：Claude Code（在项目目录中）
**前提**：AWS CLI 已配置（P-4）

```
使用 AWS CLI 部署我的 Next.js 项目到 EC2：

1. ap-southeast-1 区域创建 t3.micro 实例（Amazon Linux 2023）
2. 安全组 project-mgr-sg：开放 22, 80, 3000 端口
3. 创建 SSH Key Pair: project-mgr-key
4. SSH 连接后：安装 Node.js 22 + pnpm → 克隆 GitHub 仓库 → pnpm install && pnpm build → pm2 启动并配置开机自启
5. 输出公网 IP，验证 http://<IP>:3000 可访问

一步一步执行，每步确认后再继续。
```

### 3-2 手动部署（备选）

**在哪**：终端

```bash
# 查找最新的 Amazon Linux 2023 AMI
aws ssm get-parameters --names /aws/service/ami-amazon-linux-latest/al2023-ami-kernel-default-x86_64 --region ap-southeast-1 --query 'Parameters[0].Value' --output text

# 创建安全组
aws ec2 create-security-group --group-name project-mgr-sg --description "PM App"

# 开放端口
aws ec2 authorize-security-group-ingress --group-name project-mgr-sg --protocol tcp --port 22 --cidr 0.0.0.0/0
aws ec2 authorize-security-group-ingress --group-name project-mgr-sg --protocol tcp --port 3000 --cidr 0.0.0.0/0

# 创建 Key Pair（保存 .pem 文件）
aws ec2 create-key-pair --key-name project-mgr-key --query 'KeyMaterial' --output text > ~/.ssh/project-mgr-key.pem
chmod 400 ~/.ssh/project-mgr-key.pem

# 启动实例（用上面查到的 AMI ID 替换）
aws ec2 run-instances --image-id ami-xxx --instance-type t3.micro --key-name project-mgr-key --security-groups project-mgr-sg

# 获取公网 IP
aws ec2 describe-instances --filters "Name=key-name,Values=project-mgr-key" --query 'Reservations[0].Instances[0].PublicIpAddress' --output text

# SSH 部署
ssh -i ~/.ssh/project-mgr-key.pem ec2-user@<IP>
# 在 EC2 上执行：
curl -fsSL https://get.pnpm.io/install.sh | sh -
pnpm install && pnpm build
pnpm add -g pm2 && pm2 start pnpm --name "pm" -- start && pm2 save && pm2 startup
```

**完成后**：获得 `http://<EC2-公网IP>:3000` 在线链接，进入 Step 4
